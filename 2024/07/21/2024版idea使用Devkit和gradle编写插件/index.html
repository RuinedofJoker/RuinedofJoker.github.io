安装Devkit插件

去idea的插件市场里安装 Devkit 插件

new 插件的 Project

在new Project里的Generator里，可以看到一个 IDE Plugin选项，选择该选项
里面选择项目基本信息，其中type选Plugin，jdk版本选17以上
注意：在2024版的idea插件开发里面，使用的gradle是需要高版本的（具体哪个版本为分界线不知道，这边默认使用的是gradle-8.7），使用的jdk也要17以上

加载依赖

进入项目时默认就会开始加载依赖，我们可以先把它叉掉，把上面的配好后再点击右边gradle栏的刷新按钮再次开始加载。
需要注意的是，插件开发默认会下一个社区版的idea到gradle本地仓库里，这个加起来可能有个800Mb，而且是去idea官网下的，国内可能会很慢。我用的解决办法就是挂个代理，然后idea的settings里搜索proxy，让idea走我的代理，然后再下载。

编写插件代码

脚手架默认使用kotlin作为默认语言，所有这里默认生成了一个 kotlin 的目录。我使用的是java进行开发，所有我们把这个目录去掉，调整成java的目录结构。

把main下的kotlin目录删掉，新建一个java目录。
idea里源码的目录是会用蓝色标识出来，如果没有蓝色目录标识，就去顶栏的File -> Project Structure..里面，选择module，右边找到你刚刚新建的java源码目录，选择它，然后点击上面Mark as: 里的Sources按钮。（按理来说刚刚新建时java目录会被标记为源码目录，如果没有标记的话不做这一步会导致新建java Class文件等失效，这个是ide的问题）

这里就是常规java项目的结构了，源码目录为src/main/java，资源目录为src/main/resources。（如果你想换这两个目录的位置，去build.gradle文件里面配置，有个sourceSet项配置，具体去搜gradle相关配置）

此时你会注意到项目里面原本就有 resources 资源目录，并且里面有一个plugin.xml和一个pluginIcon.svg文件。
后者就是你插件的图标。而前者 plugin.xml 则是插件开发的一个很重要的配置文件，里面注册了你插件所有动作等的基本信息，比如你在idea里点击右键里面会不会有你插件的按钮，或者底栏有没有你插件的按钮，点击以后会以一个什么样的格式打开，类似于安卓开发的AndroidManifest.xml文件。

我们先在java源码目录里面新建你的包，然后在new Class时不要像之前一样new，我们在New的下面找到一个Plugin Devkit，里面选择Action，这时会打开一个新建action的选项菜单，我们重点填写好Action ID和Class Name，其中Action ID是需要你插件里的Action全局唯一，而Class Name则是你这个Action的类名（不是全路径类名），我们这两个可以取成一样，比如都叫MyFirstPopupMenuAction；，而Name则是你在页面上显示的名字，description是描述。下面的Add to Group决定了该Action的类型，这里我们选一个EditorPopupMenu，该类型会弹出一个弹出框。

运行或调试

这里在idea右上角运行文件选择 Run Plugin，然后点击运行或调试按钮，idea会在编译好后自动打开之前下载的idea社区版，然后你可以在里面查询插件是否生效了。
